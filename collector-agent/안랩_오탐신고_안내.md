## 2. 고객 PC 즉시 복구 (안내문 — 거래처에 보낼 메시지)

> 안녕하세요. 디직스코리아입니다.
>
> 자사 제공 프로그램(프린터카운트수집기) 이 안랩 V3 의 일시적 오탐으로
> 인해 격리·삭제된 것으로 확인됩니다. 본 프로그램은 디직스코리아이 자체
> 개발해 임대 거래처에만 제공하는 정상 프로그램이며, 안랩에 오탐 신고를
> 접수했습니다.
>
> **임시 조치 — V3 예외 등록 후 재설치**
>
> 1) 작업표시줄 우측 V3 아이콘 더블클릭 → V3 메인 창 오픈
> 2) **PC 보안** → **고급 설정** → **예외 설정** 메뉴 진입
> 3) **폴더 예외 추가**: `C:\Users\(사용자명)\AppData\Local\HanbyeolCollector` 입력 후 추가
> 4) **다운로드 폴더 예외 추가** (재설치 시 다운로드 EXE 가 다시 격리되지 않도록):
>    `C:\Users\(사용자명)\Downloads` 추가
> 5) **격리소 복원** (이미 격리된 경우):
>    V3 메인 → 보안 기록 / 격리소 → digix-collector.exe 선택 → "복원" 또는 "예외 처리"
> 6) 디직스코리아 임대거래처 페이지에서 EXE 를 다시 다운로드:
>    https://hanbyeolsystem.github.io/asms-digix/#rental-customers/index.html
>    → "고객 카운터프로그램" 버튼
> 7) 다운로드한 ZIP 압축을 풀고 안의 `digix-collector.exe` 더블클릭
>
> 안랩 오탐 처리(1~3일) 가 완료되면 위 예외 설정 없이도 정상 동작합니다.
>
> 불편을 드려 죄송합니다. 추가 문의는 053-588-7119.

## 3. 근본 대응 (디직스코리아 사내 — 이미 완료)

다음 항목으로 빌드 파이프라인을 보강했으므로 다음 배포본부터 오탐 확률이
크게 낮아집니다.

| 적용 | 효과 |
|---|---|
| `--onedir` 빌드 | EXE 가 자기 자신을 임시 폴더로 압축 해제하는 동작 제거 → V3 패커/드로퍼 의심 회피 |
| `--version-file version_info.txt` | EXE 속성에 디직스코리아·제품명·버전·저작권 노출 |
| `--manifest app.manifest` | UAC 권한(asInvoker) 명시 + DPI awareness 선언 |
| `--noupx` | UPX 패킹 명시적 배제 |
| 디직스 로고 아이콘 | 정체불명 아이콘 → 정상 SW 외관 |
| ZIP 배포 | 다운로드/실행 단계가 분리돼 일부 휴리스틱 회피 |

**남은 보강 옵션 (선택)**
- 코드 서명 인증서(예: 세인트시큐어 / DigiCert) 도입 — 연 10~30만원, 1~2주 발급
- 안랩 화이트리스트 사전 등록 신청 (대량 배포 SW 대상)
