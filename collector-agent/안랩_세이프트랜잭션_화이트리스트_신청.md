# 안랩 Safe Transaction 오탐 대응 가이드

`digix-collector.exe` 가 **AhnLab Safe Transaction (세이프 트랜잭션)** 에 의해
설치되지 않거나, 실행 중 강제 종료·삭제되는 경우 사용하는 절차서입니다.

> **주의 — V3 와 다른 제품입니다.**
> Safe Transaction 은 KB / 신한 / NH / 우리 등 주요 은행 인터넷뱅킹 접속 시
> 자동 설치되는 "금융 보안 플러그인"으로, **사용자 동의 없이 백그라운드로
> 동작**하며 통제가 매우 제한적입니다. V3 와는 별도 제품·별도 정책이라
> [[안랩_오탐신고_안내.md]] 의 V3 신고만 해서는 풀리지 않습니다.

## 2. 거래처 PC 즉시 복구 안내문 (거래처에 그대로 전달)

> 안녕하세요. 디직스코리아입니다.
>
> 자사 제공 프로그램(프린터카운트수집기) 이 **인터넷뱅킹 보안 프로그램
> (AhnLab Safe Transaction)** 의 오탐으로 설치가 되지 않거나, 설치 후
> 자동 삭제되는 사례가 일부 거래처에서 확인되었습니다.
> 현재 안랩에 화이트리스트 신청을 접수했으며 등록 완료까지 임시 조치를
> 안내드립니다.
>
> **임시 조치 절차**
>
> 1) 인터넷뱅킹 작업이 모두 끝난 상태에서 진행해주세요.
>
> 2) **Safe Transaction 종료**
>    - 작업표시줄 우측 하단 화살표(`∧`) 클릭 → 안랩 Safe Transaction 아이콘 우클릭 → "종료"
>    - 종료가 안 되면: 작업관리자(Ctrl+Shift+Esc) → 세부 정보 탭 →
>      `ASDsvc.exe`, `AhnSDsv.exe`, `ASTSrv.exe` 우클릭 → 작업 끝내기
>
> 3) **프린터카운트수집기 재설치**
>    - 사이트 접속: https://hanbyeolsystem.github.io/asms-digix/#rental-customers/index.html
>    - "고객 카운터프로그램" 다운로드 → ZIP 압축 풀기 → 안의 `digix-collector.exe` 더블클릭
>    - 설치 완료 메시지 확인 후 작업표시줄 트레이에 디직스 로고 아이콘 표시되는지 확인
>
> 4) **다음 인터넷뱅킹 접속 시 주의**
>    - 인터넷뱅킹 사이트(KB/신한/NH/우리 등) 접속하면 Safe Transaction 이
>      자동으로 다시 실행됩니다.
>    - 이때 카운트수집기가 다시 강제 종료되면 위 1~3단계 반복.
>    - 안랩 화이트리스트 처리(5~10일) 가 완료되면 위 절차 없이 정상 공존합니다.
>
> **알아두실 점**
> - Safe Transaction 은 일반 V3 백신과 다른 별도 보안 모듈입니다.
>   V3 예외 설정만으로는 풀리지 않습니다.
> - 디직스코리아 카운트수집기는 인터넷뱅킹 정보·키보드 입력·화면 등을
>   일절 수집하지 않습니다. 오로지 프린터 카운터(인쇄 매수)와 토너
>   잔량만 수집합니다. (안랩 검토용 자료 송부 완료)
