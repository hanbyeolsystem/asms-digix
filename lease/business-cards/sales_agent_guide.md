# 🚀 [시스템 구축 가이드] 영업 자동화 및 고객 관리를 위한 명함 관리 에이전트 시스템

본 문서는 명함 정보를 기반으로 **카카오톡, 이메일, 팩스 발송** 등의 영업 활동을 자동화하고, 스마트한 **영업 에이전트(Agent)**로 확장하기 위한 개발 지침 및 마스터플랜입니다. 

---

## 🗺️ 1. 시스템 아키텍처 및 로드맵

비즈니스 성공을 위해 시스템을 **3단계**로 나누어 점진적으로 구축합니다.

```
[1단계: 데이터 자산화] -> [2단계: 멀티 채널 발송] -> [3단계: AI 영업 에이전트]
- 명함 OCR & 그룹화      - 알림톡/친구톡 연동       - 고객 맞춤형 시나리오 발송
- 기존 거래처 매칭        - 이메일 & 팩스 API        - 후속 관리(Follow-up) 자동화
```

---

## 🗂️ 2. 에이전트 활동을 위한 데이터베이스(DB) 설계

영업 타겟팅(카카오톡, 메일, 팩스)을 정밀하게 수행하기 위해 확장된 DB 구조입니다.

### 1) 거래처 테이블 (`Companies`)
* `id` (PK, 정수)
* `company_name` (회사명, 인덱스 처리)
* `fax_number` (대표 팩스 번호)
* `created_at` (등록일)

### 2) 명함/고객 테이블 (`BusinessCards`)
* `id` (PK, 정수)
* `company_id` (FK, 거래처 테이블 연결 / 미분류 시 NULL)
* `name` (고객 이름)
* `mobile_phone` (휴대폰 번호 - **카카오톡 발송용**)
* `email` (이메일 주소 - **메일 발송용**)
* `fax` (직통 팩스 번호 - **팩스 발송용**)
* `status` (고객 상태: `잠재고객`, `접촉중`, `계약완료`, `이탈`)
* `last_contacted_at` (최종 연락일)

### 3) 사용자 정의 그룹 테이블 (`Groups`)
* `id` (PK, 정수)
* `group_name` (그룹명: 예 - `2026_IT전시회_잠재고객`, `VIP_관리 대상`)

### 4) 매핑 테이블 (`CardGroupRelations`)
* `card_id` (FK)
* `group_id` (FK)

## 📲 3. 영업 멀티 채널 발송 시스템 연동 지침

영업 에이전트가 고객에게 메시지를 보내기 위해 연동해야 할 핵심 API 스택입니다.

### ① 카카오톡 발송 (비즈니스 채널)
* **추천 서비스:** 카카오 비즈니스 API 파트너 (쏠라피, 인포뱅크, 비즈프리오 등)
* **구현 방식:**
    * **알림톡:** 정보성 메시지(회원가입 안내, 예약 확정 등) 전송 시 사용 (비용이 가장 저렴함).
    * **친구톡:** 영업 마케팅, 이벤트, 제안서 전송 시 사용 (고객이 내 카카오톡 채널을 추가해야 발송 가능).
    * **에이전트 활용:** 명함 등록 즉시 *"안녕하세요 [이름]님, 오늘 미팅 감사했습니다."* 라는 자동 알림톡 발송 시나리오 구축.

### ② 이메일 발송 (제안서 및 대량 메일)
* **추천 서비스:** AWS SES (Amazon Simple Email Service), Stibee (스티비) API, Mailgun
* **에이전트 활용:** 텍스트뿐만 아니라 회사 소개서(PDF) 자동 첨부 기능 구현. HTML 템플릿을 활용해 깔끔한 뉴스레터나 제안서 발송.

### ③ 팩스 발송 (전통적 B2B 영업 필수)
* **추천 서비스:** 팝빌(Popbill) Fax API, 가비아 팩스 API
* **구현 방식:** 웹사이트 내에서 PDF나 이미지 파일을 업로드하면, API를 통해 고객사 팩스 기기로 문서가 전송되도록 구현.
* **에이전트 활용:** 거래처로 묶인 그룹 전체에 공문이나 견적서를 일괄 팩스 전송 가능.

## 🤖 4. '가후의 책략' AI 영업 에이전트(Agent) 시나리오

명함 정보를 바탕으로 에이전트가 단순한 '발송 도구'를 넘어 **'비즈니스 전략가'**로 활동할 수 있도록 하는 기능적 지침입니다.

### 시나리오 1: 명함 스캔 직후의 '초전박살' 책략
1.  영업사원이 현장에서 명함을 촬영하여 업로드합니다.
2.  **AI 에이전트**가 즉시 기존 거래처 여부를 파악합니다.
    * **기존 거래처인 경우:** "기존 거래처 [A회사]의 신규 담당자입니다. 기존 영업 히스토리를 확인하세요." 알림 제공.
    * **신규 거래처인 경우:** 새 거래처로 자동 생성하고 `미분류_신규` 그룹에 할당.
3.  **자동 액션:** 등록 10분 후, 고객 휴대폰으로 **카카오톡 감사 메시지**를 자동 발송하고, 이메일로 **회사 소개서**를 자동 발송합니다.

### 시나리오 2: 그룹별 '타겟 팩스/메일 폭격' 책략
1.  영업 목적에 따라 명함들을 `[제조업_공장장_리스트]` 그룹으로 지정합니다.
2.  에이전트 대시보드에서 해당 그룹을 선택합니다.
3.  **[그룹 일괄 발송]** 버튼을 눌러, 이메일이 등록된 고객에게는 제안서 메일을, 팩스 번호만 있는 전통 비즈니스 고객에게는 제안서 팩스를 동시에 교차 발송합니다.

### 시나리오 3: '리마인드(Follow-up)' 책략
1.  에이전트가 DB의 `last_contacted_at(최종 연락일)`을 매일 스캔합니다.
2.  명함 등록 또는 마지막 연락 후 **30일 동안 소통이 없었던 고객**을 추출합니다.
3.  영업사원에게 알림을 보냅니다: *"안도현 팀장님과 연락하신 지 30일이 지났습니다. 안부 카카오톡을 보내볼까요?"* (클릭 시 미리 작성된 안부 템플릿 발송 화면으로 이동)

## 🛡️ 5. 개발 시 필수 체크리스트 (보안 및 규제)

영업 목적의 대량 발송은 법적 규제를 준수해야 에이전트 시스템이 차단되지 않습니다.

* **광고성 정보 규정 (KISA 가이드라인):** * 영업/마케팅 목적의 카카오톡, 메일, 팩스 발송 시 메시지 시작 부분에 **(광고)** 문구를 반드시 표기해야 합니다.
    * 메시지 끝부분에 무료로 수신을 거부할 수 있는 **[수신거부 번호/링크]**가 반드시 포함되어야 합니다.
* **080 수신거부 시스템 연동:**
    * 팩스 및 문자/카카오톡 발송 API 연동 시 080 수신거부 서비스도 함께 연동하여, 수신 거부한 고객은 DB에서 자동으로 발송 제외 처리되도록 로직을 짜야 합니다.
* **데이터 백업:**
    * 명함 원본 이미지와 텍스트 데이터는 영업 사원의 중요한 자산이므로 일일 백업 체계를 갖추어야 합니다.
